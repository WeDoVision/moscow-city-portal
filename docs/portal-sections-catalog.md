## Классификация по сложности (что «логика на нас», что «текст наверх»)

Ключевой принцип движка: **тяжёлая логика — в коде блока, композиция/контент — в схеме (доступно ИИ и админке)**.

### A. Контентные блоки — просто текст / кнопки / картинки

Не требуют логики. Полностью собираются из примитивов `custom`-блока
(`heading, text, image, button, divider, spacer, columns, html`) или из простых
data-less блоков (`hero, about, faq, cta`):

- Hero (фото-вариант), About + статистика, FAQ, CTA-баннер
- Образовательный лонгрид, промо-баннер, текстовые информблоки
- Подборки/локация-навигатор (плитки = картинка + текст + ссылка + счётчик)
- Footer-меню, блог-сетка (если статьи приходят готовым списком)

→ Для ИИ это «сборка из кубиков»: заголовок, абзацы, кнопка, колонки.

### B. Блоки с данными — логика фетча/состояния на нас

Тянут данные Whitewill и/или держат состояние. ИИ задаёт только пропсы
(заголовок, лимит, scope/фильтр), рендер и данные — код:

- **Фильтр + каталог объектов** — состояние фильтров, валюты, сортировка, пагинация (`lotsGrid` + фильтр-бар)
- **Карусель/полоса ЖК** (`towers` / TowersStrip)
- **Featured-объект**, карточка башни, витрина застройщиков
- **Блог** (если подтягивается из CMS), таблицы инфраструктуры (данные + сортировка/рейтинг)

### C. Интерактивные / тяжёлые блоки — реальная сложная логика

Помечены `heavy`, переиспользуют готовые client-компоненты:

- **3D / интерактивная карта района** (`city3d` → CityMap3D)
- **Фотогалерея** с фильтрами и каруселью
- **Формы заявок** — сабмит, валидация, отправка в CRM, аналитика (LeadForm)
- **Header/Footer** — навигация, избранное, мессенджеры, трекинг

---

## Ответ на вопрос «большинство секций — просто текст и кнопки?»

**Да.** Бо́льшая часть секций (категория A) — это текст, заголовки, картинки,
кнопки и колонки без собственной логики; ИИ может собирать их свободно из
примитивов. Реальная сложная логика сосредоточена в небольшом числе блоков
категории C — **каталог с фильтрами, интерактивная карта (3D), формы заявок,
галерея**. Их пишем и поддерживаем мы как «тяжёлые» блоки с фиксированным
контрактом пропсов, а ИИ/админка только размещают их и настраивают контент.
